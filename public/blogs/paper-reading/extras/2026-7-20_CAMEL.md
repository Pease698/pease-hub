## 文章概述

**研究目标**：

使用智能体完成任务往往需要精细化的指令，但人往往很难甚至不可能精准设计每一条指令以高质量提示智能体完成工作。本文希望借助智能体自动化完成指令设计这一步骤，从而提高系统的能力

**主要内容**：

![](/blogs/paper-reading/extras/CAMEL_pic/CAMEL_workflow.png)

文章提出了一个有角色扮演的多智能体系统，主要包含四个智能体：
1. Task Specifier
2. AI User
3. AI Assistant
4. Critic（可选，也可以由人类专家承担）

系统主要流程如下：
1. 用户输入简单的 idea 描述以及角色分配，包括 User Role 和 Assistant Role
	- idea 如 “为股票市场开发交易机器人”
	- User Role 如 “股票交易员”，Assistant Role 如 “python 程序员”
2. Task Specifier 将 idea 描述进行扩充，得到详细的任务描述
3. 系统使用用户指定的角色，通过硬编码 prompt 实例化两个智能体，分别作为 AI User 和 AI Assistant。两者关系为：AI User 负责根据任务描述提出指令，AI Assistant 负责生成解决方案并完成指令
	- prompt 需要强制要求智能体扮演对应的角色，同时进行其他限制以保证对话正常进行。相关内容可以查看原论文 3.2 小节和第 4 章
4. AI User 每次会根据历史对话内容（初始为详细任务描述）生成指令，AI Assistant 每次根据历史对话内容和指令生成解决方案并执行，随后将这一轮的指令和解决方案整合到历史对话中，进行下一轮循环直到达到限制条件结束
5. critic-in-the-loop 机制：
	1. 系统可以使用 prompt 实例化一个 Critic agent（也可以是人类专家担任）。prompt 中现需要有一个明确的判定标准（如成本最低、提升性能等），同时要求每次只能选择一个并给出详细理由
	2. AI Assistant 每次可以给出多个指令，并由 Critic agent 选择一个交给 AI User 执行；AI User 每次可以给出多个执行方案，由 Critic agent 选择一个进行执行。这一流程可以重复多次

论文中使用大模型自动化生成 AI User 和 AI Assistant 并创建对应的问题场景，使用 CAMEL 框架得到了四组数据集（AI Society, Code, Math, Science）。通过数据集微调基准模型以实现了更好的结果

> [!tip] 渐进式微调
> 原论文中提到了 **渐进式微调**，即逐个使用不同数据集对模型进行微调，并比较每次微调前后模型在不同方面 benchmarks 上的表现

## 启发

1. CAMEL 框架可以进行借鉴
2. 渐进式微调的实验方法可以了解
3. 除了解决具体问题，CAMEL 框架还可以用于自动化生成数据集用于模型训练
4. 在 prompt 设计中，定义模型的角色、行为边界很重要，同时也需要给出具体数据返回格式以及相关的提示词（如对话角色更换或对话终止的提示词）
