结合 AI 完成了这份深度学习基础知识的笔记

## 符号/连接/行为主义

1. **符号主义(Symbolism)：**
	- 认为智能的核心是对符号的逻辑操作。它假设世界可以被抽象为离散的符号，智能就是一套处理这些符号的规则。比如根据输入使用分支结构进行不同决策
	- **专家系统** 是符号主义的一个代表
2. **连接主义 (Connectionism)：**
	- 受生物神经网络启发，认为智能不需要预先编写显式规则，而是通过大量简单节点（神经元）的相互连接和并行运算，在海量数据中自发“涌现”出规律
	- **深度学习** 是连接主义的一个代表，通过构建多层网络和反向传播算法更新结点之间的连接，实现对复杂模式的识别
3. **行为主义 (Behaviorism)：**
	- 认为智能体必须在真实的、动态的环境中，通过不断试错来学习。它不侧重于内部知识库的构建，而是强调“感知-行动-反馈”的循环
	- **强化学习** 是行为主义的一个代表，通过进行预测后得到反馈而不断调整参数

> [!note] 专家系统
> 专家系统本质上是一个庞大且复杂的知识库和推理机，可以将其理解为编译器中的抽象语法树：规则是由人类工程师严格制定的，程序根据当前的状态输入，遍历决策树，输出确定性的结果。它的缺点是无法处理模糊的、非结构化的现实世界数据，且维护庞大规则库的代码复杂度极高。

> [!tip] 马尔可夫决策过程
> **马尔可夫决策过程 (Markov Decision Process, MDP)** 是为动态环境中的决策问题提供的一个严谨的数学框架。强化学习的所有算法几乎都是建立在 MDP 之上：
> 
> 一个 MDP 通常由一个元组 $(S, A, P, R, \gamma)$ 构成：
> - $S$：状态空间 (State)，例如棋盘当前的局面，或者内存页表的当前映射状态。
> - $A$：动作空间 (Action)，例如移动某个棋子，或者换出某个内存页。
> - $P(s' \mid s, a)$：状态转移概率，表示在状态 $s$ 执行动作 $a$ 后，转移到新状态 $s'$ 的概率。
> - $R(s, a)$：奖励函数 (Reward)，执行动作后获得的即时反馈。
> - $\gamma$：折扣因子 (Discount Factor) $\gamma \in [0, 1]$，决定了模型是短视（看重眼前奖励）还是远视（看重未来奖励）。
> 
> **马尔可夫性 (Markov Property)**：它假设**未来只取决于当前状态，而与历史路径无关**。用公式表示为：
>   $$P(s_{t+1} \mid s_t, a_t, s_{t-1}, a_{t-1}, \dots, s_0, a_0) = P(s_{t+1} \mid s_t, a_t)$$
> 
> 在实际环境中（比如复杂的机器人控制或游戏），状态转移概率 $P$ 和奖励函数 $R$ 通常是未知的。强化学习算法（如 Q-Learning、PPO）的目标，就是通过不断与环境交互，在不知道具体环境模型的情况下，求解这个 MDP，找到一个**最优策略 (Optimal Policy)** $\pi^*(s)$，告诉智能体在任何给定状态下应该采取什么动作，以最大化累计的折现奖励

> [!tip] 传统ML vs DL
> **传统机器学习（如 SVM、随机森林、XGBoost）** 适用于结构化数据（表格数据）
>
>当数据像关系型数据库一样，拥有清晰的列名和业务含义（例如年龄、收入、点击率、代码控制流图的复杂度指标），此时适合使用传统 ML。它们需要的数据量较小，训练速度快（在 CPU 上即可完成），且具备较强的可解释性
> 
> **深度神经网络 (Deep Learning)** 适用于非结构化数据（图像、音频、自然语言）
> 
> 当输入是高维且没有明显独立语义的原始数据（例如一张 $256 \times 256$ 图片里的几万个 RGB 像素，或者一段文本的 Token），传统模型会彻底失效，此时必须依靠深度学习强大的表示学习能力，在成百上千层的非线性变换中，将这些原始信号映射到有意义的隐空间（Latent Space）。它的代价是极度依赖海量数据和庞大的 GPU/TPU 算力，且存在“黑盒”不可解释的问题。

对于一个 agent，其可以同时体现这三种主义：
1. **大模型充当“大脑”（连接主义）**：负责意图理解、常识储备和复杂的语言生成
2. **外部工具充当“手脚”（行为主义）**：赋予 Agent 在环境中行动的能力（如调用计算器、执行 Python 代码、操作本地文件）
3. **规划与记忆充当“逻辑”（符号主义）**：将复杂的任务拆解为多步计划（Task Planning），并保持结构化的长期记忆

## 状态空间与搜索

### 状态空间与问题抽象

尝试将一问题抽象为一张图：
1. 图的每个结点表示当前的状态
2. 图每条边表示一个操作，可以使问题从一个状态变为另一个状态

对于大模型，同样可以将一个问题抽象为一个状态空间：
1. **状态**：一个状态可以看作一段文本表述，即当前所有已知信息
2. **动作**：动作可以分为两类：
	- **内部推理**：生成一段新的 token，如生成一段分析或假设等
	- **外部交互**：调用外部工具获取新的信息
3. **状态转移**：若进行外部推理，则相当于将新的文本添加到上下文中；若调用外部工具，则相当于将调用结果添加到上下文中
4. **目标**：可以使用大模型基于当前上下文给出评估

### 搜索算法

若使用广度优先 BFS，需要在运行时记录当前层的所有状态；若使用深度优先 DFS，则难以找到最短路径，且可能陷入无限循环中（如图中存在环）

此时可以使用启发式算法，定义启发函数 $h(n)$ 以估算当前结点到目标结点的距离，从而有针对性的向目标进行搜索。一种启发式搜索为 $A^*$ 算法，该算法在每一步评估节点的总代价：

$$f(n) = g(n) + h(n)$$

其中：
- $g(n)$：从起点走到当前节点 $n$ 的实际已知代价
- $h(n)$：从当前节点 $n$ 走到终点的预估代价（启发直觉）
- A* 算法每次都挑选 $f(n)$ 最小的节点进行探索，既保证了能找到最优解，又极大地减少了不必要的搜索范围。

> [!note] 思维树 ToT
> ToT 适合需要多步严密逻辑、但步骤不至于无限深的任务（如复杂数学证明、架构设计）
> 1. **步骤拆解 (Decomposition)**：首先通过 Prompt 要求 LLM 将大问题拆解为 $K$ 个独立的思考步骤
> 2. **生成节点 (Thought Generation)**：在每个步骤，不要让 LLM 只给出一个想法，而是设置 `temperature > 0.7`，让它独立生成 3-5 个不同的候选思路（即在树上分叉）
> 3. **状态评估 (State Evaluation)**：这是 ToT 的核心。针对生成的 5 个思路，你需要发起额外的 LLM API 请求，让它自己评估这些思路的前景，通常分类为：`Sure/Likely/Impossible`
> 4. **搜索策略 (Search Algorithm)**：结合 BFS（广度优先）或 DFS（深度优先）。比如，扔掉所有被评为 `Impossible` 的分支，只在评为 `Sure` 的分支上继续往下生成下一步的思路

> [!note] 蒙克卡洛树 MCTS
> MCTS 适合那些“搜索空间极大，且只有走到最后一步才知道结果是否正确”的复杂工程问题。如现在需要检查 QEMU 下运行的 RISC-V 内核的指令页错误：
> 1. **选择 (Selection)**：Agent 观察当前的 C/汇编源码状态以及 GDB 捕获的寄存器上下文，在已有排查方向中，利用上限置信区间算法 (UCT) 选择一个兼顾“高成功率”和“未充分探索”的分支。
> 2. **扩展 (Expansion)**：大模型针对当前的报错提出几个具体的修复猜想（动作）。比如：
> 	- 猜想 A：“Sv39 页表的 PTE 权限位设置错误。”
> 	- 猜想 B：“satp 寄存器的模式字段未正确切换。”
> 3. **模拟推演 (Simulation / Rollout)**：Agent 根据猜想 A，自动生成补丁代码，调用外部工具重新编译，并在 QEMU 环境中实际运行一次。由于模型具有强大的代码生成能力，这个“推演”不仅是逻辑推导，而是**真实的工程验证**。
> 4. **反向传播 (Backpropagation)**：评估推演结果。如果修改后依然触发相同的 Page Fault，得分为 0；如果故障地址发生变化，或者成功进入了下一个内核态中断，给予正向奖励。将这个得分沿着树结构反向传回根节点，更新“猜想 A”的胜率。
> 5. **循环迭代**：随着 MCTS 的迭代，Agent 会自动放弃那些导致编译失败或死机的无效方向，最终收敛到能够成功引导内核通过页表初始化的那条代码修改路径。

> [!tip] 贪婪/束搜索
> **贪婪搜索**：每次生成时，仅选择概率最大的那种可能
> 
> **束搜索**：每次生成时同时考虑概率最大的 K 种可能，对这些可能并行验证

## 知识和推理

- 事实：客观存在的已知条件，如当前状态或公理
- 规则：基于领域知识总结出的条件判断逻辑，通常呈现 IF-THEN 形式
- 将规则应用到事实上以得到新事实的过程，包含前向推理（条件推结论）和后向推理（结论反推条件）

当 Agent 拥有知识图谱或严格的逻辑规则库时：
1. 事实校验：Agent 在生成答案前，可以通过查询知识图谱（这是确定的、不可篡改的实体关系）来验证自己的回答
2. 降低搜索空间：在进行复杂问题的 ReAct（推理与行动）时，结构化知识可以帮 Agent 剪枝，告诉它哪些 API 调用或参数组合在逻辑上是绝对不合法的，从而避免盲目试错

> [!note] 知识图谱
> 构建知识图谱的核心在于“知识表示”，即如何用计算机可计算的数据结构来描绘现实世界。主要的实现与表示方式如下：
> - **三元组 (Triples - 资源描述框架 RDF)**：知识图谱最原子的信息单元。它将世界强制解构为`(主语, 谓语, 宾语)`或`(实体, 关系, 实体)`的有向边结构
> - **属性图 (Property Graph)**：在三元组的基础上，允许节点（实体）和边（关系）本身携带一系列键值对（属性）。这是目前工业界图数据库（如 Neo4j）最广泛采用的底层数据模型，极大增强了数据表达的丰富度
> - **本体论 (Ontology - OWL)**：充当知识图谱的“元数据”或“数据字典”。它不是用来存具体数据的，而是用来定义数据的层级结构（比如实体类型的继承关系）以及强逻辑约束（比如传递性、对称性）。本体层使得图谱具备了机器推理的能力
> - **底层物理存储**：为了保证遍历效率，知识图谱通常放弃传统关系型数据库的表连接 (Join) 机制，转而采用原生图数据库。图数据库在内存中实现了“免索引邻接”，让指针直接指向关联节点，从而实现毫秒级的多跳逻辑关系查询

> [!note] 检索增强 RAG
> 检索增强生成（RAG）旨在解决大语言模型幻觉和知识过时的问题，其核心在于如何高效地存储和检索外部知识。目前有两种主要的结合路径：
> - **向量 RAG (Vector RAG)**：主要依赖高维向量空间进行存储。将非结构化文本切块，通过 Embedding 模型转化为稠密向量并存入向量数据库。检索时，计算用户查询向量与知识库向量的余弦相似度。这种方式擅长捕捉“语义相关性”，但在处理跨文档的复杂因果关系或多跳逻辑链条时容易丢失上下文
> - **图 RAG (GraphRAG)**：主要依赖图数据结构进行存储。先利用信息抽取技术，将文本解析为结构化的实体和关系，构建出知识图谱。检索时，基于用户意图在图谱上进行拓扑遍历和子图提取。图 RAG 能够提供极其精确的逻辑关联和全局结构上下文，有效弥补了向量检索在复杂逻辑推理上的短板
> 
> 现代 Agent 架构通常采用**混合检索策略**：用向量检索快速定位核心实体节点，再利用图检索沿该节点提取逻辑关系网，最后合并输入给大模型

## 机器学习基本范式

三种机器学习类型：
1. **监督学习 (Supervised Learning)**
    - **核心逻辑**：给模型提供“带标准答案”的数据（即数据 $X$ 和标签 $Y$）。模型的目标是学习一个映射函数 $f(X) \to Y$
    - **适用场景**：明确知道想要预测什么，如：
        - **分类 (Classification)**：预测离散类别（例如：输入一张猫的图片，输出标签“猫”）
        - **回归 (Regression)**：预测连续数值（例如：输入房屋面积和地段，预测具体房价）
2. **无监督学习 (Unsupervised Learning)**
    - **核心逻辑**：给模型提供“没有标准答案”的数据（只有 $X$，没有 $Y$）。模型的目标是自行发现数据内部隐藏的结构、模式或分布规律
    - **适用场景**：面对海量未知数据，需要先进行探索性分析。如：
        - **聚类 (Clustering)**：将相似的数据自动分堆（例如：根据用户的浏览行为自动将用户分为不同画像群体）
        - **降维 (Dimensionality Reduction)**：在尽量不丢失信息的前提下，剔除冗余特征，将高维数据压缩到低维
3. **强化学习 (Reinforcement Learning)**
    - **核心逻辑**：没有静态的数据集，而是让智能体（Agent）在动态环境（Environment）中通过不断试错来学习。Agent 根据当前状态采取行动，环境反馈奖励（Reward）或惩罚。模型的目标是最大化长期累积奖励
    - **适用场景**：需要连续决策的动态控制问题（例如：自动驾驶、机器人控制、AlphaGo 下棋）

数据集可以分为：训练集、验证集、测试集。机器学习时有两种数据遍历类型：
- **Batch（批次）**：每次仅从训练集中随机抽取一小部分数据（例如 32、64 或 256 个样本）输入模型，计算一次误差并更新一次参数
- **Epoch（轮次）**：当所有的 Batch 都被模型完整地“看”过一遍时，称为完成了一个 Epoch。训练通常会循环成百上千个 Epoch

训练的基本过程为：
- **训练集 (Training Set)**：模型日常做练习题的过程
    - 前向传播计算结果，对比真实标签产生 Loss（误差），然后反向传播计算梯度，**实际更新模型的内部参数（即权重 Weight 和偏置 Bias）**
- **验证集 (Validation Set)**：相当于日常的模拟考试。模型**绝对不能**通过验证集来更新内部参数（不能偷看模拟考答案）
    - **机制**：在训练集跑完一个 Epoch（或特定的步数）后，让模型在验证集上做一次预测，计算出验证误差 (Validation Loss)
    - **影响的参数**：验证集的结果决定了**超参数 (Hyperparameters)** 的调整。超参数是人类开发者设定的参数，例如：网络应该有几层、学习率 (Learning Rate) 是多少、Batch Size 设置多大、何时应该提前停止训练等
- **测试集 (Test Set)**：相当于最终的高考。它对模型和开发者都是完全未知的
    - **机制**：在所有超参数都调好、模型完全定型之后，只在测试集上运行**一次**，得出的准确率就是该模型在真实环境中的泛化能力，直接写在论文或项目报告中

模型训练涉及的三个概念：
- **损失函数 (Loss Function)**：衡量模型当前预测值 $\hat{y}$ 与真实值 $y$ 之间差距的数学函数
- **优化 (Optimization)**：一种寻找最小化损失函数的方法，通过计算损失函数对每个参数（权重和偏置）的偏导数，算法知道应该把哪个参数调大一点、哪个调小一点，才能让整体误差下降
- **泛化 (Generalization) 与 过拟合/欠拟合**：
    - **泛化能力**：模型在**测试集**（没见过的数据）上表现优异的能力，这是机器学习的终极目标
    - **欠拟合 (Underfitting)** 即模型太简单，连训练集的规律都没学明白
    - **过拟合 (Overfitting)** 即模型太复杂，把训练集里的“噪声”和“特例”都死记硬背下来了，导致一遇到新数据就彻底崩溃

> [!note] 损失函数
> 常用的损失函数包括：
> 1. **均方误差 (MSE - Mean Squared Error)**
> 	   - **原理**：预测值与真实值差值的平方和的平均数。公式为 $\text{MSE} = \frac{1}{N} \sum_{i=1}^{N} (y_i - \hat{y}_i)^2$
> 	   - **适用**：回归任务（如预测温度、价格）
> 2. **交叉熵损失 (Cross-Entropy Loss)**
>     - **原理**：衡量两个概率分布之间的差异。预测的概率分布越接近真实的标签分布（通常是 One-hot 编码），交叉熵越小
>     - **适用**：分类任务（如图像识别、文本分类）。它在梯度计算时能有效避免 Sigmoid 等激活函数的梯度消失问题

> [!note] 优化方法
> 常用的优化方法包括：
> 1. **随机梯度下降 (SGD - Stochastic Gradient Descent)**
>     - **原理**：每次仅根据当前 Batch 计算出的梯度方向，按设定的步长（学习率）反向走一步
>     - **特点**：最基础，但在复杂的地形（如鞍点）中容易陷入局部最优或震荡
> 2. **动量优化 (Momentum)**
>     - **原理**：模拟物理学中的惯性。在 SGD 的基础上，保留一部分上一次更新的方向
>     - **特点**：如果在同一方向上持续下降，速度会越来越快；如果遇到震荡地形，惯性能帮助抵消掉来回的震荡，加速收敛
> 3. **Adam (Adaptive Moment Estimation)**
>     - **原理**：目前深度学习中最默认、最常用的优化器。它结合了动量机制，并且能**自适应地为每个参数分配独立的学习率**
>     - **特点**：收敛速度极快，对超参数（如初始学习率）的选择不那么敏感，适用于绝大多数场景

> [!note] 防止过拟合
> 部分防止模型过拟合的方法：
> 1. **提前停止 (Early Stopping)**
>     - **操作**：在每个 Epoch 结束后记录验证集 Loss。如果训练集 Loss 还在下降，但验证集 Loss 连续 $N$ 轮不降反升，说明模型开始“死记硬背”了，立刻强行终止训练，并保存验证集表现最好的那一轮模型。
> 2. **权重衰减 (L1 / L2 正则化)**
>     - **操作**：在损失函数后面加上一个“惩罚项”，依据模型权重的大小进行惩罚。
>     - **效果**：L2 正则化会迫使所有权重尽可能小且均匀（限制模型复杂度）；L1 正则化会迫使很多无用特征的权重变成绝对的 0（实现特征选择）。
> 3. **Dropout**
>     - **操作**：在训练过程中，按设定的概率（例如 50%）**随机让网络中的部分神经元“宕机”**（输出置为 0），每次 Batch 屏蔽的神经元都不同。但在测试时恢复全员工作。
>     - **效果**：打破了神经元之间相互依赖的“共适应性”，强迫每个神经元独立提取有用的特征，就像团队协作中防止几个人摸鱼一样。
> 4. **数据增强 (Data Augmentation)**
>     - **操作**：在数据送入模型前，对其进行随机扰动。
>     - **效果**：如果是图像，可以进行随机裁剪、翻转、加噪声、改变对比度；如果是文本，可以进行同义词替换。通过人为扩充数据多样性，让模型见识更多变体，从而提高泛化能力。

## 强化学习

**策略** 为从状态到动作的映射。它回答了“如果在状态 $s$，我该采取什么动作 $a$”。分为确定性策略（直接输出一个动作）和随机性策略（输出各种动作的概率分布）

**价值函数** 包含 **状态价值函数 $V(s)$** 和 **动作价值函数 $Q(s,a)$**，分别评估一个状态和动作的奖励

在强化学习过程中，智能体会面临两种选择：
- **利用 (Exploitation)**：智能体完全信任当前的价值函数，每次都选择那个 $Q(s,a)$ 最高的动作
- **探索 (Exploration)**：智能体以一定概率（通常用参数 $\epsilon$ 表示）放弃当前看似最优的动作，随机选择一个动作

一个例子是使用 **$\epsilon$-greedy 算法**：设定一个随时间衰减的探索率 $\epsilon$
- 在训练初期，$\epsilon$ 很大，智能体会进行大量随机探索
- 随着价值函数逐渐收敛，$\epsilon$ 逐渐减小，智能体开始转为“利用”，只在极少数情况下进行探索

> [!tip] 贝尔曼方程
> 强化学习的目标是最大化未来总收益，但我们无法在一开始就计算得到每一步的结果。**贝尔曼方程** 使用动态规划思想将这一问题拆解为当前状态的即时奖励和下一个状态的长期价值的递归组合
> 
> 以动作价值函数（Q 函数）为例，其最优贝尔曼方程 (Bellman Optimality Equation) 的核心形式如下：
> $$Q^*(s, a) = \mathbb{E} \left[ R_{t+1} + \gamma \max_{a'} Q^*(s_{t+1}, a') \right]$$
> - $Q^*(s, a)$：在状态 $s$ 下采取动作 $a$ 的最优预期总收益
> - $R_{t+1}$：采取动作 $a$ 后环境给出的**即时奖励**
> - $\gamma$：**折扣因子 (Discount Factor)**，介于 0 到 1 之间。表示未来的奖励不如眼前的奖励实在
> - $\max_{a'} Q^*(s_{t+1}, a')$：在下一个状态 $s_{t+1}$ 中，能够选出的**最佳动作的长期收益**
>   
> 智能体在环境中不断学习和尝试，收集 $(s, a, R, s')$ 的数据片段。每收集到一个片段，就套用上面的公式，把等式右边计算出的“目标值”更新左边的 $Q(s, a)$。经过成千上万次的迭代，整个状态空间的 Q 值就会像水波纹一样，从目标点向外扩散并收敛

> [!note] 强化学习算法
> 1. **Q-Learning (基于价值)**：它不直接学习策略，而是去求解并维护一个 Q 表（或用神经网络拟合的 Q 网络）。这个网络输入状态 $s$ 和动作 $a$，输出一个具体的分数。其策略为：遇到状态 $s$，计算所有动作的 Q 值，选择分数最高的动作执行
> 2. **Policy Gradient (基于策略)**：它跳过了价值评估这一步，直接用神经网络去拟合“策略”本身。网络输入状态 $s$，输出的是**所有动作的概率分布** $\pi_\theta(a\vert{}s)$。训练时，如果某个动作带来了高额奖励，就通过计算梯度（Gradient Ascent）去增大输出这个动作的概率。
> 3. **PPO (Proximal Policy Optimization，近端策略优化)**：这是目前 OpenAI 等主流机构最常用的 Policy Gradient 进阶算法。传统的 PG 算法因为一次更新导致策略彻底崩坏，无法收敛。PPO 引入了**截断机制 (Clipping)**，强制规定每次模型参数更新的幅度不能超过某个阈值（比如比原策略变化不超过 20%），从而保证训练稳定性
> 
> 在使用中，Q-Learning 适用于“离散且动作空间小”的问题，而 PPO 适用于“超大动作空间或连续动作空间”。由于大模型对齐训练的动作空间规模很大，所以不使用 Q-Learning 算法

> [!note] 置信区间上界 UCB
> 传统的 $\epsilon$-greedy 算法在探索时是“瞎蒙”的（纯随机）。而 UCB 认为，我们应该优先去探索那些**我们目前最不确定的选项**
> 
> UCB 在做决策时，不会仅仅看当前的平均收益，而是会计算一个指标，选择该指标最大的动作：
> $$A_t = \arg\max_{a} \left[ Q_t(a) + c \sqrt{\frac{\ln t}{N_t(a)}} \right]$$
> 这个公式由两部分组成，绝妙地平衡了探索与利用：
> - **$Q_t(a)$（利用项）**：当前对动作 $a$ 评估的平均收益。如果这个动作过去表现很好，这部分数值就大，算法倾向于继续利用它
> - $c \sqrt{\frac{\ln t}{N_t(a)}}$**（探索项 / 置信区间上限）**：
>     - $N_t(a)$ 是动作 $a$ **已经被选择过的次数**。它在分母上
>     - $t$ 是**总共进行决策的次数**
>     - 如果一个动作很少被选择（$N_t(a)$ 很小），分母极小，整个探索项就会暴增。这就相当于给这个未知的动作加了一个巨大的“好奇心滤镜”，强迫算法去尝试它
>     - 随着某个动作被不断尝试，我们对它的认知越来越确定（$N_t(a)$ 增大），探索项趋近于 0，此时算法退化为纯粹的“利用”，完全依靠真实的 $Q_t(a)$ 来做决定

## 线性/Softmax 回归

线性回归假设输入特征与输出目标之间存在线性关系。数学表达式为 $\hat{y} = \mathbf{w}^T \mathbf{x} + b$，使用的损失函数一般为均方误差函数

Softmax 回归用于多类别分类，其过程描述如下：

首先，数据特征向量 $\mathbf{x}$ 经过线性层映射，得到各个类别的原始得分向量 $\mathbf{z}$（通常称为 Logits）。假设有 $K$ 个类别：

$$z_i = \mathbf{w}_i^T \mathbf{x} + b_i \quad (i = 1, 2, \dots, K)$$

这里的 $z_i$ 可以是任意实数（如 10.5, -3.2）

为了把毫无限制的 Logits 变成概率分布，数学上必须满足两个强制条件：
1. **非负性**：每个类别的概率必须大于等于 0。
2. **归一化**：所有类别的概率总和必须等于 1。

由此，Softmax 函数的数学形式为：
$$\hat{y}_i = \frac{e^{z_i}}{\sum_{j=1}^K e^{z_j}}$$
其中，$\hat{y}_i$ 就是模型预测输入 $\mathbf{x}$ 属于第 $i$ 个类别的概率

有了预测概率分布 $\hat{\mathbf{y}}$ 和真实的标签分布 $\mathbf{y}$（通常是 One-hot 向量，如 `[1, 0, 0]`），我们需要一个损失函数来衡量它们的差异。Softmax 的损失函数常使用交叉熵：

$$L = -\sum_{i=1}^K y_i \log(\hat{y}_i)$$

当在反向传播中对交叉熵损失计算关于 Logits $z_i$ 的偏导数时，复杂的指数和对数会相互抵消，最终得到一个极其优雅的梯度公式：

$$\frac{\partial L}{\partial z_i} = \hat{y}_i - y_i$$

也就是说，**梯度 = 预测概率 - 真实标签**。模型只需要根据这个简单的误差信号去更新权重即可

## 多层感知机与表示学习

### 多层感知机 (Multilayer Perceptron, MLP)

多层感知机是最基础的深度神经网络架构，通常也被称为全连接神经网络 (Fully Connected Neural Network)

- **核心结构**：在输入层和输出层之间，加入了一个或多个**隐藏层 (Hidden Layers)**。每一层的每一个神经元都与上一层的所有神经元相连（这就是全连接的由来）
- **数学本质**：如果网络只有一个隐藏层，它的前向传播可以表示为一系列矩阵运算的组合：
    $$
	\begin{aligned}
    \mathbf{h} &= \sigma(\mathbf{W}^{(1)}\mathbf{x} + \mathbf{b}^{(1)})\\
    \mathbf{y} &= \mathbf{W}^{(2)}\mathbf{h} + \mathbf{b}^{(2)}
	\end{aligned}
    $$
    其中 $\mathbf{x}$ 是输入，$\mathbf{W}$ 和 $\mathbf{b}$ 是权重和偏置，$\sigma$ 是激活函数，$\mathbf{h}$ 是隐藏层输出，$\mathbf{y}$ 是最终预测结果

> [!tip] 隐藏层
> 简单的线性模型（如逻辑回归）只能解决线性可分的问题。引入隐藏层后，网络在理论上被证明具备**通用近似定理 (Universal Approximation Theorem)**，即只要隐藏层足够宽，MLP 可以拟合任何复杂的连续函数

如果没有激活函数，无论叠加多少隐藏层，连续的矩阵乘法在数学上都可以等效折叠成一个单层的线性变换（因为 $\mathbf{W}^{(2)}(\mathbf{W}^{(1)}\mathbf{x}) = (\mathbf{W}^{(2)}\mathbf{W}^{(1)})\mathbf{x} = \mathbf{W}'\mathbf{x}$）。激活函数的引入打破了这种线性叠加，使得网络能够学习高度复杂的非线性模式

> [!note] 激活函数
> 常见的激活函数包括：
> 1. **Sigmoid 函数**：
>     将任意实数映射到 $(0, 1)$ 区间，早期常用于输出概率
>     $$f(x) = \frac{1}{1 + e^{-x}}$$
> 2. **Tanh (双曲正切) 函数**：
>     将任意实数映射到 $(-1, 1)$ 区间。它的均值为 0，在优化时比 Sigmoid 更容易收敛
>     $$f(x) = \frac{e^x - e^{-x}}{e^x + e^{-x}}$$
> 3. **ReLU (Rectified Linear Unit) 函数**：
>     现代深度学习的基石。保留正数，将负数置零。计算极快，且有效缓解了梯度消失
>     $$f(x) = \max(0, x)$$
> 4. **Leaky ReLU 函数**：
>     为了解决 ReLU 中负数部分梯度完全为零（神经元坏死）的问题，赋予负半轴一个极小的斜率（如 $\alpha = 0.01$）
>     $$f(x) = \max(\alpha x, x)$$

> [!tip] 权重初始化
> 如果将网络所有权重初始化为 0，反向传播时所有神经元的梯度都一样，网络失去了打破对称性的能力。随机初始化是必须的，但随机的方差极大影响了前向传播的方差和反向传播的梯度稳定性
> 1. **Xavier 初始化 (Glorot Initialization)**：
>    - **目标**：保持输入和输出的方差一致，避免信号在前向传递时爆炸或消失
>    - **公式**：从均值为 0，方差为 $\sigma^2 = \frac{2}{n_{in} + n_{out}}$ 的正态分布或均匀分布中采样（$n_{in}$ 和 $n_{out}$ 为该层输入输出神经元数）
>    -  **适用**：主要针对线性激活函数或对称激活函数（如 Tanh、Sigmoid）
> 2. **Kaiming 初始化 (He Initialization)**：
>    - **目标**：由于现代网络多用 ReLU 激活函数，ReLU 会强行砍掉一半的负信号，导致方差减半。Xavier 初始化在 ReLU 网络中会导致信号逐步消失
>    - **公式**：补偿被 ReLU 砍掉的方差，从均值为 0，方差为 $\sigma^2 = \frac{2}{n_{in}}$ 的正态分布中采样
>    - **适用**：专为 ReLU 及其变体设计

### 正则化

**正则化(Regularization)** 是所有用来**惩罚模型复杂度、防止过拟合**的技术统称。在深度学习中，它主要体现为以下几种工程手段：
1. **L1 / L2 正则化 (权重衰减 Weight Decay)**：
    在损失函数后附加一个惩罚项。L2 惩罚项 $\lambda \sum w^2$ 会迫使网络的权重变得极其平滑且接近于 0；L1 惩罚项 $\lambda \sum \vert{}w\vert{}$ 会迫使部分无用特征的权重绝对为 0，产生稀疏矩阵
2. **Dropout**：
    深度学习专属的正则化“大杀器”。在每次前向传播时，随机让隐藏层中一定比例（如 50%）的神经元输出变为 0。这迫使网络不能过度依赖某几个特定的神经元，而是将特征分散到全局，类似于一种极限的集成学习
3. **提前停止 (Early Stopping)**：
    监控验证集的 Loss，当训练集 Loss 下降但验证集 Loss 开始反弹时，强制终止训练

### 前向传播和反向传播

把神经网络看作是一个有向无环图。在这个图中，节点是数学操作，边是流动的数据。在这个框架下：
- **正向传播**就是数据从输入端流入，顺着计算图的边，一步步执行数学运算，最终输出预测结果和误差的过程
- **反向传播**则是将这个误差信号沿着计算图逆流而上，利用微积分的链式法则，算出每个参数对最终误差的“责任大小”（即梯度），从而知道该如何调整参数

下面以一个包含单层隐藏层的多层感知机为例，进行数学推导，假设网络的结构如下：
- 输入特征：$\mathbf{x}$
- 隐藏层权重与偏置：$\mathbf{W}^{(1)}, \mathbf{b}^{(1)}$
- 隐藏层激活函数：$\sigma$ (例如 ReLU 或 Sigmoid)
- 输出层权重与偏置：$\mathbf{W}^{(2)}, \mathbf{b}^{(2)}$
- 预测输出：$\hat{\mathbf{y}}$
- 真实标签：$\mathbf{y}$
- 损失函数：$L$

**正向传播 (Forward Propagation) —— 计算预测与误差**

正向传播是一个纯粹的数值代入和前向计算过程，本质上是大量乘加运算（MAC）和非线性映射的堆叠。

**第 1 步：计算隐藏层的线性输出（Pre-activation）**

输入数据乘以隐藏层权重并加上偏置：

$$\mathbf{z}^{(1)} = \mathbf{W}^{(1)}\mathbf{x} + \mathbf{b}^{(1)}$$

**第 2 步：计算隐藏层的激活输出（Activation）**

将线性结果送入非线性激活函数，进行特征的表示学习：

$$\mathbf{a}^{(1)} = \sigma(\mathbf{z}^{(1)})$$

**第 3 步：计算输出层的线性输出**

隐藏层的输出作为输出层的输入，再次进行线性变换：

$$\mathbf{z}^{(2)} = \mathbf{W}^{(2)}\mathbf{a}^{(1)} + \mathbf{b}^{(2)}$$

**第 4 步：计算最终预测结果**

根据具体任务，对输出层进行激活（如果是多分类，这里通常是 Softmax 函数）：

$$\hat{\mathbf{y}} = \mathbf{a}^{(2)} = \text{Softmax}(\mathbf{z}^{(2)})$$

**第 5 步：计算损失（Loss）**

利用交叉熵或均方误差，计算预测值与真实标签的差异：

$$L = \text{Loss}(\hat{\mathbf{y}}, \mathbf{y})$$

> [!tip]
> 在系统执行正向传播时，必须要在内存（SRAM或显存）中缓存所有的中间结果（$\mathbf{x}, \mathbf{z}^{(1)}, \mathbf{a}^{(1)}, \mathbf{z}^{(2)}$）。这些缓存被称为 Activation Memory，因为接下来的反向传播需要用到它们。这在芯片架构设计中是一个核心的显存占用瓶颈

**反向传播 (Backward Propagation) —— 计算梯度**

反向传播的目标是求出损失函数 $L$ 对网络中所有可学习参数（$\mathbf{W}^{(1)}, \mathbf{b}^{(1)}, \mathbf{W}^{(2)}, \mathbf{b}^{(2)}$）的偏导数（梯度），以便使用梯度下降法更新它们。这完全依赖于**链式法则 (Chain Rule)**

为了推导方便，定义一个核心变量：**误差项 (Error Term) $\boldsymbol{\delta}^{(l)}$**，它表示损失函数对第 $l$ 层线性输出 $\mathbf{z}^{(l)}$ 的偏导数，即 $\boldsymbol{\delta}^{(l)} = \frac{\partial L}{\partial \mathbf{z}^{(l)}}$

**第 1 步：计算输出层的误差项 $\boldsymbol{\delta}^{(2)}$**

这是反向传播的起点。如果使用 Softmax 加交叉熵损失，数学上恰好会发生极其优雅的抵消，结果极其简单：

$$\boldsymbol{\delta}^{(2)} = \frac{\partial L}{\partial \mathbf{z}^{(2)}} = \hat{\mathbf{y}} - \mathbf{y}$$

**第 2 步：计算输出层参数的梯度**

利用链式法则 $\frac{\partial L}{\partial \mathbf{W}^{(2)}} = \frac{\partial L}{\partial \mathbf{z}^{(2)}} \cdot \frac{\partial \mathbf{z}^{(2)}}{\partial \mathbf{W}^{(2)}}$。因为 $\mathbf{z}^{(2)} = \mathbf{W}^{(2)}\mathbf{a}^{(1)} + \mathbf{b}^{(2)}$，所以：

$$
\begin{aligned}
\frac{\partial L}{\partial \mathbf{W}^{(2)}} &= \boldsymbol{\delta}^{(2)} (\mathbf{a}^{(1)})^T\\
\frac{\partial L}{\partial \mathbf{b}^{(2)}} &= \boldsymbol{\delta}^{(2)}
\end{aligned}
$$

**第 3 步：误差信号回传到隐藏层（计算 $\boldsymbol{\delta}^{(1)}$）**

现在要把输出层的误差 $\boldsymbol{\delta}^{(2)}$ 通过权重矩阵 $\mathbf{W}^{(2)}$ “反向推”回隐藏层

利用链式法则 $\boldsymbol{\delta}^{(1)} = \frac{\partial L}{\partial \mathbf{z}^{(1)}} = \left( \frac{\partial \mathbf{z}^{(2)}}{\partial \mathbf{a}^{(1)}} \right)^T \frac{\partial L}{\partial \mathbf{z}^{(2)}} \odot \frac{\partial \mathbf{a}^{(1)}}{\partial \mathbf{z}^{(1)}}$：

$$\boldsymbol{\delta}^{(1)} = (\mathbf{W}^{(2)})^T \boldsymbol{\delta}^{(2)} \odot \sigma'(\mathbf{z}^{(1)})$$

_(注：$\odot$ 表示哈达玛积，即对应元素逐个相乘。$\sigma'(\mathbf{z}^{(1)})$ 是激活函数的导数。如果这里导数极小，就会发生梯度消失现象)_

> [!tip] 梯度消失
>如果使用 Sigmoid 作为激活函数，其导数的最大值仅为 0.25。当有一个 10 层的网络时，梯度回传到最浅层，需要乘以 10 次 0.25（即 $0.25^{10} \approx 0.0000009$）
> 
> 这会导致误差信号在反向传递的过程中呈指数级衰减。到达最浅层时，梯度已经近似变成 0，最浅层的权重无法更新，这正是深层网络在 ReLU 发明前难以训练的根本原因

**第 4 步：计算隐藏层参数的梯度**

同样利用链式法则，根据算出的 $\boldsymbol{\delta}^{(1)}$ 和正向传播时缓存的输入 $\mathbf{x}$，得出最浅层参数的梯度：

$$
\begin{aligned}
\frac{\partial L}{\partial \mathbf{W}^{(1)}} &= \boldsymbol{\delta}^{(1)} \mathbf{x}^T\\
\frac{\partial L}{\partial \mathbf{b}^{(1)}} &= \boldsymbol{\delta}^{(1)}
\end{aligned}
$$

**第 5 步：参数更新**

所有层的梯度计算完毕后，优化器（如 SGD 或 Adam）接管，执行实际的参数修改：

$$\mathbf{W}^{(l)} \leftarrow \mathbf{W}^{(l)} - \eta \frac{\partial L}{\partial \mathbf{W}^{(l)}}$$

### 表示学习

表示学习是指机器**自动从原始数据中发现并提取有用特征**的过程。在传统机器学习中，算法工程师需要花大量时间进行“特征工程”（比如手动设计滤波器提取图像边缘）。而在深度学习中，这一步被网络内部的隐藏层接管了

- **浅层**：提取底层的、局部的像素特征（如边缘、颜色梯度）
- **中层**：将浅层特征组合成有具体形态的部件（如车轮、猫的耳朵）
- **深层**：将中层部件组合成高维的全局语义表示（如“一辆汽车”、“一只猫”）。 网络通过反向传播，为了最小化最终的分类误差，**自动且逐层**地寻找到了对解决当前任务最完美的内部数据表达形式

## 卷积神经网络

### 卷积

#### 卷积定义

**卷积(Convolution)** 是通过两个函数 $f$（输入）和 $g$（卷积核）生成第三个函数 $h$（特征图/Feature Map）的数学算子：
$$h(t) = \int g(t-t')f(t)dt'$$
而在深度学习最常见的多维离散情况下，无限的积分退化为有限数组的累加：
$$H(i,j) = \sum_{m} \sum_{n} F(m,n)G(i-m, j-n)$$

> [!tip] 
> 在数学定义中，我们需要把系统的响应函数 $h(t)$ 沿时间轴翻转变成 $h(-t)$。这是因为最先发生的输入信号，到现在已经经历最长的时间，其衰减或响应处于函数的最末端

**互相关（Cross-Correlation）**：在深度学习（如 PyTorch、TensorFlow）的实际代码实现中，通常省去将卷积核翻转这一步。因为卷积核内部的参数是不断通过反向传播训练更新的，有没有这步翻转对最终的特征表达没有任何实质上的影响。因此，深度学习中的“卷积”在严格数学定义上其实是**互相关（Cross-Correlation）**：
$$H(i,j) = \sum_{m} \sum_{n} F(m,n)G(i+m, j+n)$$

#### 卷积与全连接对比

CNN 是一种基于卷积操作的神经网络，可以粗略地理解为 CNN 就是将全连接网络中的矩阵相乘换成卷积操作（说粗略是因为通常 CNN 中还带有 pooling 等 CNN 专有的操作）

通过将一个具体的 $2 \times 2$ 卷积核与一个 $3 \times 3$ 输入矩阵的计算进行展开，可以看到卷积与全连接网络的相同和区别

如果把 $3 \times 3$ 的输入矩阵拉平（Flatten）为一个 $9 \times 1$ 的向量，那么卷积计算得到的 $2 \times 2$ 输出（同样拉平为 $4 \times 1$ 向量）可以用一个大矩阵乘法来等效表示：
$$\begin{bmatrix} 4 & 6 & 0 & 7 & 9 & 0 & 0 & 0 & 0 \\ 0 & 4 & 6 & 0 & 7 & 9 & 0 & 0 & 0 \\ 0 & 0 & 0 & 4 & 6 & 0 & 7 & 9 & 0 \\ 0 & 0 & 0 & 0 & 4 & 6 & 0 & 7 & 9 \end{bmatrix} \begin{bmatrix} 43 \\ 16 \\ 78 \\ 34 \\ 76 \\ 95 \\ 35 \\ 8 \\ 46 \end{bmatrix} = \begin{bmatrix} 4 \times 43 + 6 \times 16 + 7 \times 34 + 9 \times 76 \\ \dots \\ \dots \\ 4 \times 76 + 6 \times 95 + 7 \times 8 + 9 \times 46 \end{bmatrix}$$
对比全连接网络（Fully Connected NN）的参数矩阵：
- **全连接的特点**：其参数矩阵（例如上面的 $4 \times 9$ 权重矩阵）里所有位置都是非零的独立变量。这意味着每个输出神经元都负责连接输入的所有像素。参数量极其庞大，计算量和内存需求高，且难以训练
- **卷积的两大先验限制**：
    1. **稀疏连接 (Sparse Connectivity)**：在等效的核矩阵中，有大量的元素被强制填零（例如只有 4 个位置有值）。这意味着每个输出神经元只关注输入图像的一个局部
    2. **参数共享 (Parameter Sharing)**：在非零位置上，参数值是完全重复且一样的（都是由 $4, 6, 7, 9$ 构成的卷积核）

这种先验限制是建立在“高维空间中相邻的数据点存在一定关系”这一物理基础之上的，虽然加入这种硬编码的先验限制会让模型损失一定的自由度（理论拟合能力下降），但它大大减少了参数数量，并在多个不同位置复用同一套参数，极有利于捕获局部的平移不变特征。最终效果上，这种限制带来的泛化和抗过拟合收益远远大于拟合能力的微小损失

#### 三种卷积模式

根据卷积核移动的起始和终止位置，以及输出特征图的尺寸不同，深度学习框架通常提供三种经典的卷积模式。以 5x5 输入，3x3 卷积核，步长 Stride = 1 为例：
1. **FULL 模式**
    - **规则**：只要卷积核与输入图像存在哪怕一个点的相交，就开始进行卷积计算
    - **表现**：这需要我们在输入的四周补上大量的零（Padding = 2）。因为滑动的边界最宽，其移动区域最大，因此**卷积后输出的尺寸也最大**（输出尺寸为 $7 \times 7$）
2. **VALID 模式**
    - **规则**：与 FULL 模式相反，要求整个卷积核完全与输入区域重叠才开始计算
    - **表现**：该模式**不需要任何 Padding**（Padding = 0）。因为滑动的边界受到了严格限制，其移动区域最小，因此**输出的尺寸也最小**（输出尺寸为 $3 \times 3$）
3. **SAME 模式**
    - **规则**：最常用的一种模式。它的目标是让卷积后输出的尺寸与输入的尺寸保持完全一致（假定 Stride = 1）
    - **表现**：通过将卷积核的物理中心点与输入图像的第一个像素对齐来确定起始位置，并在边缘自动补齐对应大小的 Padding
    - **特殊情况（偶数核）**：当卷积核的边长为偶数（如 $2 \times 2$）时，为了让输出和输入尺寸仍旧保持一致，系统会在其中一边（行或列）多增加一行 Padding，通过**不对称的 Padding** 来强制实现输出尺寸与输入尺寸的绝对相等（如 $5 \times 5 \to 5 \times 5$）

### 卷积神经网络 CNN

相比全连接神经网络，卷积神经网络存在以下核心概念和操作：
1. **卷积**
2. **局部感受野 (Local Receptive Field)**：CNN 中的神经元不再像 MLP 那样连接整张图，而是只关注前一层的一小块局部区域（例如 $3 \times 3$ 像素）。层数越深，感受野的累积范围越大，从而实现从“局部边缘”到“全局语义”的层次化抽象
3. **池化 (Pooling)**：最常用的是最大池化 (Max Pooling)。它在局部窗口内只保留最大值，直接对特征图进行降采样（宽高减半）。池化剔除了冗余信息，并赋予了网络**平移不变性**（目标稍微移动几个像素，依然能被识别）
4. **填充 (Padding)**：卷积核在图像边缘滑动时，会导致输出图像变小。为了保持输出尺寸一致，通常在输入图像边缘补零（Zero-padding）
5. **步长 (Stride)**：卷积核每次滑动的像素距离。步长为 2 时，输出特征图的尺寸会直接缩小一半

> [!note] 池化 Pooling
> 
> 
> 池化，在信号处理领域通常被称为**降采样（Downsampling）**。在经历过卷积操作后，我们会得到一张特征图（Feature Map）。池化的作用就是拿一个固定大小的窗口（通常是 $2 \times 2$）在特征图上滑动，每次只保留窗口内最具代表性的一个值，从而将特征图的尺寸缩小（通常宽高各减半）
> 
> 最常用的池化有两种：
> 1. **最大池化（Max Pooling）**：取窗口内的**最大值**。（最常用，因为能保留最强烈的纹理和边缘特征）
> 2. **平均池化（Average Pooling）**：取窗口内所有值的**平均值**。（早期常用，能够保留背景等全局信息）
> 
> > [!example]
> > 假设我们经过一次卷积后，得到了一张 $4 \times 4$ 的特征图，里面的数字代表了某个特征（比如“猫耳朵”）在当前位置的激活强度。数字越大，说明该位置存在“猫耳朵”的概率越高
> > 
> > 我们使用一个 $2 \times 2$ 的池化窗口，并且设置步长（Stride）为 2（这意味着窗口每次滑动跨越 2 个像素，窗口之间完全不重叠）
> > 
> > **原始输入 ($4 \times 4$ 矩阵)：**
> > $$\begin{bmatrix} 4 & 6 & 1 & 3 \\ 7 & 9 & 2 & 8 \\ 2 & 1 & 5 & 2 \\ 0 & 4 & 1 & 3 \end{bmatrix}$$
> > **最终输出 ($2 \times 2$ 矩阵)：**
> > $$\begin{bmatrix} 9 & 8 \\ 4 & 5 \end{bmatrix}$$
> 
> 引入池化操作的好处：
> 1. **局部平移不变性（Translation Invariance）**：池化赋予了网络对微小位移、轻微扭曲的容忍度，让网络只关心“特征存在与否”，而不过度苛求“特征精确在哪个像素点”
> 2. **降低内存与算力开销**：如果插入一个 $2 \times 2$ 的池化层，特征图的宽和高会直接减半，意味着下一层的数据量和计算量直接缩减为原来的四分之一 (25%)。这在极大缓解显存压力的同时，也降低了过拟合的风险（因为特征变少了）
> 3. **扩大后续层的“感受野”（Receptive Field）**：池化通过缩小特征图，变相地让深层网络感受野扩大，从而能够从宏观上整合局部的边缘特征，识别出复杂的全局物体

> [!tip] im2col
> 二维图像的像素在逻辑上是相邻的，但在物理内存中是以一维线性连续存储的（Row-major），会导致频繁发生 CPU/GPU 的 Cache Miss
> 
> **im2col (Image to Column)** 优化逻辑为：
> 1. **数据重排**：它将卷积核在输入图像上每一次滑动所覆盖的局部感受野（比如一个 $3 \times 3$ 的图块），“拉平”成一维列向量
> 2. **构建大矩阵**：随着滑动窗口遍历整张图，所有这些列向量拼成了一个巨大的二维矩阵。这确实会导致内存占用变大（由于滑窗重叠，部分像素被复制了多次）
> 3. **转化为 GEMM**：此时，原本复杂的嵌套循环滑窗，被彻底转化为一次标准的通用矩阵乘法 (GEMM)，从而可以利用底层对 GEMM 的优化提升性能
>  
>  ![](/blogs/deeplearning-note-fundamentals/im2col.png)

> [!tip] 图卷积神经网络 GCN
> 很多 3D 对象都是由顶点 (Vertices) 和拓扑边 (Edges) 构成的不规则多边形网格 (Mesh)
> 
> 对于这类对象，一种方法是使用 3D 体素网格进行重构并使用 3D 卷积核进行卷积操作以构建 3D CNN。但这种方法的计算复杂度较高，且 3D 模型仅包含表层的体素信息，内部空间全部为 0，这样做会浪费大量算力
> 
> **图卷积神经网络, Graph Convolutional Network** 将 Mesh 的顶点视为图的“节点 (Node)”，边视为“连接”。GCN 的卷积不再是“滑窗”，而是**消息传递 (Message Passing)**：每个顶点通过拓扑边，主动收集并聚合其一阶邻居顶点的特征。这样可以原生地捕捉网格在变形、拉伸下的局部几何特征关系，且不会造成算力浪费

### CNN 架构演进

1. **LeNet-5 (1998)**：
    - **特点**：CNN 的奠基之作，确立了 `卷积层 -> 池化层 -> 全连接层` 的标准范式
    - **应用**：早期主要用于银行支票的邮政编码（手写数字）识别。网络极浅（5 层有效参数层），使用 Sigmoid/Tanh 作为激活函数，参数量小
2. **AlexNet (2012)**：
    - **特点**：引爆了现代深度学习浪潮。相比 LeNet，它更深（8 层）、更宽
    - **核心创新**：首次大规模引入了 **ReLU 激活函数**（有效缓解梯度消失）、**Dropout**（防止大规模参数过拟合），并开创性地利用多块 GPU 并行计算来突破算力瓶颈
3. **ResNet (残差网络, 2015)**：
    - **特点**：解决了“网络退化”难题。在此之前，网络堆叠到 20 层以上时，训练误差不仅不降反而急剧上升
    - **核心创新**：引入了**跳跃连接 (Skip Connection)**，构建了残差块。网络不再直接拟合完整映射 $H(x)$，而是拟合残差 $F(x)$，最终输出为 $F(x) + x$。这个 $+x$ 可以让底层梯度可以毫无衰减地反向传回浅层，使得训练上百层甚至上千层的超深网络成为现实

> [!note] ResNet
> 在多层网络中，每一层都会不可避免地引入计算噪声、丢失信息或者发生梯度消失，导致网络层数很大时会发生 **网络退化问题**
> 
> ResNet 架构中引入了 **跳跃连接**。假设某一个网络块（Block）的输入是 $x$，原本需要这个块的内部网络拟合一个完美的输出 $H(x)$。有了跳跃连接后，直接把输入 $x$ 绕过这个块输出上。此时，内部网络不再需要拟合完整的 $H(x)$，而是去拟合一个残差（Residual） $F(x)$。最终的输出变成了：
> $$\text{Output} = F(x) + x$$
> 这种设计可以使得某层网络的输出和梯度都能保持较好的效果

### 传统 CV 对比大模型

- **CNN 的归纳偏置 (Inductive Bias)**：传统 CV 使用 CNN。CNN 预设了人类对视觉世界的理解即“相邻像素关联强（局部性）”和“平移不变性”。这种强预设使得 CNN 在小规模数据集上学习效率极高，下限很高
- **ViT (Vision Transformer)**：ViT 将图像切分成多个无重叠的 Patch（像语言模型里的 Token），直接丢给 Transformer 架构，利用**全局自注意力机制 (Self-Attention)** 计算图像中任意两块区域的关联
- **为什么 ViT 上限远超 CNN？** 由于 ViT 抛弃了局部性的预设，它在数据量少时表现不如 CNN（容易过拟合）。但随着数据量和算力扩展到亿级别，CNN 的“强预设”反而成了束缚；而 ViT 能够直接从海量数据中自发学习出极其复杂的、跨越整张图像的长距离空间依赖关系。在当前的超大规模多模态模型中，ViT 的上限和泛化能力是 CNN 无法企及的

## 序列建模与注意力机制

### 循环神经网络 RNN

**序列数据** 不仅有内容，而且数据的 **先后顺序** 包含着核心信息。打乱顺序，数据的语义就会彻底改变或者被破坏

传统的神经网络是“无状态”的，每次输入互相独立。而 RNN（循环神经网络）是一个**有状态系统（Stateful System）**，相当于在网络中加入了一个隐式寄存器（隐藏状态 $h_t$），用来缓存历史信息的累加

假设输入序列为 $X = [x_1, x_2, \dots, x_T]$，则在单层 RNN 中：
- **正向传播 (Forward Propagation)**：
    在时间步 $t$，RNN 接收当前的输入 $x_t$ 和上一时刻的隐藏状态 $h_{t-1}$
    1. **状态更新**：$h_t = \tanh(W_{xh} x_t + W_{hh} h_{t-1} + b_h)$
    2. **输出计算**：$y_t = W_{yh} h_t + b_y$
- **反向传播 (BPTT, Backpropagation Through Time)**：
    RNN 的反向传播需要把网络在时间轴上“展开（Unroll）”。当我们计算损失函数 $L$ 对权重矩阵 $W_{hh}$ 的梯度时，必须对所有时间步的梯度求和：
    $$\frac{\partial L}{\partial W_{hh}} = \sum_{t=1}^T \frac{\partial L_t}{\partial W_{hh}}$$
    而要算 $\frac{\partial L_t}{\partial W_{hh}}$，必须利用链式法则一直追溯到初始时刻：
    $$\frac{\partial L_t}{\partial W_{hh}} = \frac{\partial L_t}{\partial y_t} \frac{\partial y_t}{\partial h_t} \sum_{k=1}^t \frac{\partial h_t}{\partial h_k} \frac{\partial h_k}{\partial W_{hh}}$$
    核心难点就在于这个**跨越时间的偏导数连乘** $\frac{\partial h_t}{\partial h_k}$，容易发生梯度消失或梯度爆炸

> [!tip] 梯度消失/爆炸
> **梯度消失**：
> - 早期时间步的梯度几乎为零，模型无法学习长距离依赖。即“记不住太久远的信息”
> - 缓解办法：
> 	- 使用 **LSTM** 或 **GRU** 等门控结构，通过“加法”式的信息流切断连乘，使梯度可以无损远距离传播
> 	- 更好的参数初始化（如正交初始化 $W_{hh}$​）
> 	- 使用 ReLU 配合恰当初始化可稍缓解，但普通 RNN 用 ReLU 风险较大，容易发散
> 
> **梯度爆炸**：
> - 参数更新步长巨大，损失剧烈震荡甚至溢出 NaN
> - 缓解办法：**梯度裁剪**（Gradient Clipping）是标准方案，将梯度的 $L_2$​ 范数限制在阈值内

多层（堆叠）RNN 是把多个 RNN 层垂直堆叠起来。第 $l$ 层的隐藏状态序列，会作为第 $l+1$ 层的输入序列。假设共有 $L$ 层，其计算过程如下（省略激活函数）：
- **第 1 层**：$h_t^{(1)}=f^{(1)}(x_t,\;h_{t-1}^{(1)})$
- **第 2 层**：$h_t^{(2)}=f^{(2)}(h_t^{(1)},\;h_{t-1}^{(2)})$
- ...
- **第 $L$ 层**：$h_t^{(L)}=f^{(L)}(h_t^{(L-1)},\;h_{t-1}^{(L)})$

这里 $f^{(l)}$ 表示第 $l$ 层的循环单元，拥有独立的权重 $W^{(l)}$

> [!tip]
> 在同一层 RNN 中，参数 $W_{xh},\;W_{hh}$ 均共享，只有在不同的 RNN 层中才相互独立

和图像卷积神经网络类似，高层 RNN 更能感知到输入的高层特征（如语义等），底层 RNN 更倾向于感知局部特征

> [!tip] 双向 RNN
> 时刻 $t$ 的输出不仅取决于过去，也依赖未来。双向 RNN 运行两条独立的 RNN：一条正向处理 $x_1\rightarrow x_T$​，一条反向处理 $x_T\rightarrow x_1$，然后将二者的隐藏状态拼接起来用于预测

### QKV 计算框架

给定一个 Query 向量 $q$，以及一组 Key-Value 向量对 $((k_1, v_1), (k_2, v_2), \dots, (k_n, v_n))$，注意力机制的计算步骤如下：

**1. 计算注意力得分（相似度）**

得分函数 $f$ 有多种选择，最常见的是**点积**和**缩放点积**：
$$
score(q, k_i) = q \cdot k_i \quad \text{或} \quad \frac{q \cdot k_i}{\sqrt{d_k}}
$$其中 $d_k$ 是 Key 向量的维度。除以 $\sqrt{d_k}$ 是为了防止点积值过大，导致 softmax 梯度过小而难以训练

**2. 归一化为概率分布（注意力权重）**

通过 softmax 将所有得分转化为 $[0, 1]$ 之间的权重，且和为 1：
$$
\alpha_i = \frac{\exp(\text{score}(q, k_i))}{\sum_{j=1}^n \exp(\text{score}(q, k_j))}
$$
权重 $\alpha_i$ 越大，代表第 $i$ 个 Value 对本次 Query 越重要

**3. 聚合 Value（输出）**

最终的注意力输出是所有 Value 向量的加权和：
$$
\text{Attention}(q, K, V) = \sum_{i=1}^n \alpha_i v_i
$$

> [!tip]
> 
> 实际的神经网络都是批量、矩阵化地处理序列。将所有向量堆叠成矩阵后，计算极其简洁。假设有 $n$ 个 Key-Value 对，维度分别为 $d_k$ 和 $d_v$。有 $m$ 个 Query
> - 矩阵 $Q$：$m \times d_k$
> - 矩阵 $K$：$n \times d_k$
> - 矩阵 $V$：$n \times d_v$
> 
> **缩放点积注意力**可一步完成：
> $$
> \text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right) V
> $$
> 这里的 softmax 按行进行（每个 Query 的权重独立归一化）。矩阵乘法的计算效率极高，这也是 Transformer 能高效训练的关键

### 注意力机制

注意力机制不需要按照时间序列的顺序进行逐步计算，而是在处理输入时“环顾”整个序列中的所有其他数据，并动态决定把“注意力”集中在哪些相关性最高的部分。注意力机制有以下几种主要形式：
1. **交叉注意力 (Cross-Attention)**：常用于 Encoder-Decoder 架构（如机器翻译）。模型一边看着源语言（提供 Key 和 Value），一边生成目标语言（提供 Query）
2. **自注意力 (Self-Attention)**：它完全发生在序列内部，序列自己既是查询者，又是被查询者
3. **因果注意力 (Causal/Masked Attention)**：用于 GPT 等 Decoder-only 模型。在预测下一个词时，强制屏蔽（Mask）掉未来的词，只允许看到当前和过去的词
4. **多头注意力 (Multi-Head Attention)**：这是对以上机制的增强。将 $Q, K, V$ 用 $h$ 组不同的投影矩阵并行映射到较小的子空间，分别计算注意力，最后将输出拼接： $$\text{MultiHead}(Q, K, V) = \text{Concat}(\text{head}_1, \dots, \text{head}_h) W^O$$每个“头”可以学习关注不同的模式（比如一个头专注位置，一个头专注语义），使模型容量和表达能力大幅提升

> [!note] 自注意力机制
> 在 Self-Attention 中，序列的输入特征矩阵 $X$ 会分别乘以三个独立的可学习权重矩阵（$W_Q, W_K, W_V$），在**同一个数据源**上投影出三个不同的分身：
> - $Q = X W_Q$
> - $K = X W_K$
> - $V = X W_V$
> 
> 然后执行 Attention 公式：$Attention(Q, K, V) = \text{Softmax}(\frac{QK^T}{\sqrt{d_k}})V$
> 
> 这相当于让序列中的每一个 Token 自己生成搜索条件（Q），自己生成标签（K），然后去搜索序列内部的其他 Token（V）

> [!tip] FlashAttention
> 在注意力机制中，当计算 Attention 时若序列长度过大会导致 SRAM 中无法完全存储，只能借助速度很慢的 HBM 进行存储，此时会导致速度大大降低
> 
> FlashAttention 借助下面三个方面提升训练速度：
> 1. 分块 (Tiling)
> 	将 $Q,K,V$ 分割成许多能塞进 SRAM 的小块。计算注意力时，只从 HBM 加载当前所需的 $Q_i$​ 和 $K_j$​ 块，在芯片内计算局部注意力，直接累加到输出 $O_i$​ 块上。整个过程，完整的 $N\times N$ 矩阵从未在 HBM 中实体化。
> 2. 在线 Softmax (Online Softmax)
> 	标准 Softmax 需要三遍遍历：找最大值（减它防溢出）、指数求和、归一化。分块后，每块的数据是独立的，无法一次性获得全局信息
> 	FlashAttention 采用了一种惰性修正的并行流水线方式。它在循环分块时，维护两个标量状态：
> 	- $m$：当前已发现的行最大值
> 	- $ℓ$：当前已累计的指数和
> 	当处理一个新块 $K_jV_j$​ 时，若发现更大的值 $m_{new}$​，它会动态地重新缩放之前已计算好的部分输出，以确保最终结果与完整矩阵一次性 Softmax 在数学上严格等价
> 3. 重计算 (Recomputation)
> 	训练时需要反向传播，但正向计算产生的 $N\times N$ 注意力矩阵不再被存储。反向传播时，所需的 $S$ 和 $P$ 矩阵值直接从 SRAM 中的 $Q,K,V$ 子块快速重新计算

## Transformer 基础架构

### transformer 整体结构

transformer 采用了经典的“编码器-解码器”架构，包含 encoder 和 decoder 两个部分：
1. **编码器 (Encoder)**：
	- 负责“理解”。它读取完整的源语言句子，利用自注意力机制，将每一个词转换为包含全局上下文信息的高维特征向量
    - 这就像是一个特征提取流水线。由于 Encoder 不需要考虑“未来”的信息限制它可以一次性把整个句子吃进去，所有词的特征提取是完全并行计算的
2. **解码器 (Decoder)**：
    - 负责“生成”。它接收 Encoder 提取出的全局特征，并结合自己目前已经生成的词，自回归地（Autoregressively，即一个词接一个词地）预测下一个词
    - Decoder 中包含了一种特殊的**掩码自注意力 (Masked Self-Attention)**。在生成第 $t$ 个词时，硬件会通过在注意力矩阵上强加一个掩码（填入负无穷大），强制屏蔽掉 $t$ 时刻之后的词，防止模型“偷看”未来的答案

![](/blogs/deeplearning-note-fundamentals/TransformerArch.png)

虽然原始 Transformer 采用 Encoder-Decoder 结构，但后续发展出了多种变体——**仅编码器**（如 BERT，擅长文本理解）、**仅解码器**（如 GPT，擅长文本生成）。当前的大语言模型（LLM）主流采用的是 **Decoder-only（仅解码器）架构**

### 多头注意力机制

使用多头注意力机制可以并行关注多个维度的信息。将原始的高维词向量（比如 512 维）切分成 $h$ 个低维的子空间（比如 8 个头，每个头 64 维）。在每个子空间里，独立初始化一套 $W_Q, W_K, W_V$ 权重，分别进行 Attention 计算。最后将这 8 个头的输出拼接（Concatenate）回 512 维，再做一次线性映射即可得到输出

以 768 维词向量和 12 个子空间（每个空间 64 个维度）为例，多头注意力划分的步骤如下：

**第一步：线性投影（不划分，先升维）**  

输入矩阵 $X$ 分别乘以三个权重矩阵 $W_Q,W_K,W_V​$（由不同子空间的权重矩阵组合得到），得到 $Q,K,V$。此时它们的维度依然是 `[序列长度, d_model]`（即 768 维）

**第二步：张量重塑（真正划分维度）**  

代码中会执行一个 **`view()`**（重塑）操作，这是划分的关键：
- 将 `[序列长度, 768]` 重塑为 `[序列长度, 12, 64]`
- 此时，768 个特征维度被物理地切分成了 12 组，每组包含 64 个维度
- 接着进行维度交换（`permute`），将“序列长度”和“头数”换位，变成 `[12, 序列长度, 64]`，方便后续并行计算

**第三步：独立计算**  

这 12 组 `[序列长度, 64]` 分别独立进行缩放点积注意力计算（$Attention(Q,K,V)$）。这 12 个计算过程完全并行，且权重矩阵（$W_Q,W_K,W_V$​）各自独立、互不干扰

计算完 12 个头后，需要把结果拼回 768 维：
1. 将 12 个头的输出拼接（Concat）：`[12, 序列长度, 64]` → `[序列长度, 768]`
2. 乘以一个最终的输出权重矩阵 $W_O$​，完成线性变换

> [!tip] 输出权重矩阵
> 如果最后没有输出权重矩阵 $W_O$，仅将子空间结果拼接后输出，会存在几个问题：
> 1. 子空间结果之间非常割裂，不同子空间的维度之间不存在任何联系
> 2. 后续还有残差连接步骤，若简单拼接后加上本层输出进行后续操作，会导致语义空间上的不匹配
> 
> 输出权重矩阵 $W_O$ 为一个全连接矩阵，对输入进行线性变换后可以使每个输出维度都融合所有头的输入维度信息，从而避免上面的问题

### 位置编码

自注意力机制本身不考虑输入的顺序。为了让模型能够感知到词的位置信息，引入 **位置编码**：
1. **绝对位置编码**（需要将位置编码向量和词向量相加）：
	1. 正弦/余弦编码
	2. 可学习位置编码
2. **相对位置编码**
	1. 旋转位置编码（Rotary Position Embedding, RoPE）
	2. 带线性偏差的注意力（Attention with Linear Biases, ALiBi）

**正弦/余弦编码**：

这是2017年原始Transformer论文提出的经典方案。其精妙之处在于，它并非学习一个位置向量，而是用不同频率的正弦和余弦函数直接计算出来

对于位置 `pos` 和维度索引 `i`，其编码值为：

$$
\begin{aligned}
PE(pos, 2i) &= sin(pos / 10000^{(2i/d_{model})})\\
PE(pos, 2i+1) &= cos(pos / 10000^{(2i/d_{model})})
\end{aligned}
$$
- **唯一且有界**：不同频率的正弦/余弦波组合，保证了每个位置的编码都是唯一的，且值域在 $[-1, 1]$ 之间
- **蕴含相对位置信息**：通过三角函数的恒等变换，位置 `pos+k` 的编码可以被位置 `pos` 的编码通过一个线性变换表示出来。这让模型能轻易捕捉到词语间的相对距离
- **优点**：无需额外参数，且理论上可以处理任意长度的序列
- **缺点**：外推能力弱。当推理时的序列长度远超训练时，编码会进入未训练的周期区间，导致性能下降

**可学习位置编码**：

另一种常见方法是，不预设任何数学公式，而是将位置编码当作**可训练的参数**，让模型在训练过程中自己“学习”出最好的位置向量
- **实现方式**：维护一个形状为 `[max_len, d_model]` 的可训练矩阵。输入时，直接取出对应位置的向量与词向量相加
- **优点**：灵活，理论上能更好地适应特定任务的数据分布
- **缺点**：**泛化能力受限**。因为矩阵大小是固定的（`max_len`），一旦测试序列长度超过训练时设定的最大值，模型就无参数可用
- **典型应用**：原始的 BERT、GPT 等模型都采用了可学习的位置编码

> [!tip]
> 绝对位置编码会将位置编码向量和词向量直接相加，但这并不会破坏原有的语义信息
> 
> Transformer 内部的向量维度极高（例如 512 维，甚至 4096 维）。在概率论与高维几何中存在定理：**在极高维空间中，随机生成的两个向量，或者代表完全不同特征（语义 vs 位置）的向量子空间，大概率是相互垂直（正交）的**
> 
> Attention 的核心是算两个词的点积（Query 和 Key 的相似度）
> 
> 假设词 A 的输入是 $X_A = W_A + P_A$（词义 A + 位置 A），词 B 的输入是 $X_B = W_B + P_B$（词义 B + 位置 B）
> 
> 当模型计算 A 和 B 的点积时有：
> $$
> \begin{aligned}
> X_A \cdot X_B^T &= (W_A + P_A) \cdot (W_B + P_B)^T\\
> X_A \cdot X_B^T &= \underbrace{W_A W_B^T}_{\text{词义与词义}} + \underbrace{W_A P_B^T}_{\text{词义与位置}} + \underbrace{P_A W_B^T}_{\text{位置与词义}} + \underbrace{P_A P_B^T}_{\text{位置与位置}}
> \end{aligned}
> $$
> 1. **$W_A W_B^T$**：比较两个词在**纯语义**上是否相关（比如“苹果”和“水果”得分高）
> 2. **$P_A P_B^T$**：比较两个词在**物理距离**上是否相关（比如相邻的词得分高）
> 3. **交叉项**：通常用于学习特定的句法规则（比如“第 1 个位置的词通常是主语”）
> 
> 在训练过程中，模型内部的投影矩阵 ($W_Q, W_K$) 会自动学习分配权重，独立地在这四项中提取需要的信息。**相加操作不仅没有破坏信息，反而让模型可以用一套最简单的线性点积，同时并行处理“语义”和“距离”两种信号**

**旋转位置编码**：

**RoPE（Rotary Position Embedding）** 是目前最主流、最先进的方案，被 LLaMA、Qwen、DeepSeek 等绝大多数现代大模型采用
- **核心思想**：它不把位置编码加到词向量上，而是通过**旋转矩阵**，直接对 Query 和 Key 向量进行“旋转”操作。位置越靠后的词，其Q和K向量被旋转的角度就越大
- **如何工作**：
    1. 对于位置为 `m` 的词，其Query向量 `q` 会被乘以一个旋转矩阵 `R_m`
    2. 当计算位置 `m` 和 `n` 的两个词的注意力分数时，实际上是计算 `(R_m * q)` 和 `(R_n * k)` 的点积
    3. 这个点积的结果，天然地只取决于 `m` 和 `n` 的差值，也就是它们的相对距离
- **优点**：
    - **强大的外推能力**：通过旋转角度的线性增长，模型能更好地泛化到比训练时更长的序列
    - **无额外参数**：旋转矩阵是数学计算得出的，不需要额外的可训练参数

**带线性偏差的注意力**：

ALiBi（Attention with Linear Biases）采用了一种更直接、更暴力的方法
- **核心思想**：它彻底抛弃了在词向量上加位置编码的做法。而是在计算注意力分数时，直接**减去一个与两个词之间距离成正比的惩罚项**
- **如何工作**：对于距离为 `d` 的两个词，它们的注意力分数会直接减去 `m * d`（`m` 是一个超参数）。距离越远，惩罚越大，注意力分数就越低
- **优点**：实现极其简单，且几乎没有计算开销
- **缺点**：超参数 `m` 需要针对不同模型和任务进行调试
- **典型应用**：在 MPT-7B 等模型中得到了应用

### Encoder

Encoder 由若干结构完全相同的 **编码器块 Encoder Block** 串联而成，输入矩阵分别经过这些编码器块后得到输出

#### 文本输入

1. **第一步：分词与数字化 (Tokenization)**
    一段文本首先会被分词器（Tokenizer，如 BPE 或 WordPiece 算法）切分成一个个 Token，随后根据预先训练好的词表（Vocabulary），将每个 Token 映射为一个整数 ID
- **第二步：词嵌入查表 (Word Embedding)**
    模型内部维护着一个巨大的词嵌入矩阵 $E$（大小为 $\text{词表大小} \times \text{模型维度} D$），模型会用上一步得到的整数 ID 作为索引，去这个矩阵里抽出对应的行向量。此时，原本离散的词变成了一个 512 维的稠密浮点数向量。这个向量包含了词的**静态基础语义**
- **第三步：注入位置编码 (Positional Encoding)**
    模型会生成一个与词向量维度完全相同的**位置向量**（通过正弦/余弦公式计算，或通过学习得到），最后将词向量与位置向量进行**逐元素相加 (Element-wise Addition)**。若为相对位置向量则按对应的方式进行操作

经过这三步，文本才真正变成了 Encoder 可以计算的输入矩阵 $X$（$序列长度 \times 特征维度$）

#### 编码器块

一个编码器块可以分为两部分：多头注意力层(Multi-Head Self-Attention) 和前馈神经网络层(Feed-Forward Network)，其中每一层后都会有残差连接和层归一化运算

**多头注意力层**

进行线性投影、注意力打分以及结果拼接后进行输出

这一阶段可视为针对每个词都将其他词的信息根据相关性收集并整合到自己身上，作为后续的输入

**前馈神经网络层**

由两层全连接层和中间一层激活函数组成，数学公式为：
$$
\text{FFN}(x) = \max(0, xW_1 + b_1)W_2 + b_2
$$
- **第一层线性变换（升维）**：将输入维度 $d_{model}$（例如 768）映射到一个极高的中间维度 $d_{ff}$（通常是 $d_{model}$ 的 4 倍，即 3072）。这一层负责将注意力层提取的特征“展开”到高维空间，挖掘更丰富的特征组合
- **激活函数（非线性）**：原始论文使用 **ReLU**（即公式中的 $\max(0, ...)$），作用是引入非线性。现在主流大模型（如LLaMA）已改用更平滑的 **GELU** 或 **SwiGLU** 变体
- **第二层线性变换（降维）**：将高维空间（3072）投影回原来的维度 $d_{model}$（768），作为该子层的最终输出

FFN 是“位置逐点”（Position-wise）运行的，对于输入序列中的每一个词，FFN都会独立地对其进行相同的数学变换，不同词之间不会相互干扰

可以将 FFN 层看作一个存储着海量“模式-补全”对的大型记忆库

- **第一层权重 $W_1$：充当“键”（Key）检测器**。$W_1$ 的每一行（或每一列，取决于视角）可以被看作是一个“键”。当输入一个词，它会与所有这些“键”进行匹配。某个“键”被激活的程度越高，就说明当前输入与它存储的模式越匹配
- **第二层权重 $W_2$​：充当“值”（Value）存储器**。与 $W_1$​ 中的“键”相对应，$W_2$ 中存储着“值”。当一个“键”被激活后，模型就会根据激活强度，去 $W_2$​ 中提取对应的“值”，并将它们加权组合，形成FFN层的最终输出

这种设计使得模型能够“记住”在训练数据中频繁共现的模式，并在遇到类似输入时“回忆”起来

**残差连接**

将某个子层（比如 MHA 或 FFN）的原始输入矩阵与其输出矩阵进行逐元素相加
$$
Y = X + \text{Sublayer}(X)
$$

**层归一化(LayerNorm)**

对于输入矩阵中的每一个词向量 $x$（长度为 $D$），独立计算这 $D$ 个数字的均值 $\mu$ 和方差 $\sigma^2$。然后对这 $D$ 个数字进行标准化，最后乘以可学习的缩放参数 $\gamma$ 并加上平移参数 $\beta$
$$
\begin{aligned}
\hat{x} &= \frac{x - \mu}{\sqrt{\sigma^2 + \epsilon}}\\
\text{LayerNorm}(x) &= \gamma \hat{x} + \beta
\end{aligned}
$$
    
充当数值稳压器。经过 MHA 的加权求和以及 FFN 的放大后，特征向量内部的数值往往会变得极大或极小（方差剧烈波动）。LayerNorm 强行把每个词的特征拉回到“均值为 0，方差为 1”的标准状态

> [!tip]
> 在 encoder 部分中，有以下四部分参数需要学习：
> 1. 词嵌入矩阵参数，位置编码矩阵（如果有）
> 2. 多头注意力层参数
> 3. 前馈神经网络层参数
> 4. 层归一化的缩放因子和平移因子
> 
> 训练过程同样为四个步骤：前向传播、损失计算、反向传播、优化器更新

### Decoder

#### Decoder 数据流

Decoder block 包含三个部分：
1. **掩码多头自注意力 (Masked Multi-Head Self-Attention)**
2. **编码器-解码器交叉注意力 (Encoder-Decoder Cross-Attention)**
3. **前馈神经网络 (FFN)**

Decoder 进行自回归生成内容时：
1. 每次会将当前已生成的内容进行词嵌入和位置编码后作为第一层掩码多头自注意力层的输入
2. 第一层输出会和 Encoder 输出一起输入到第二层编码器-解码器交叉注意力层
3. 第二层输出作为第三层前馈神经网络的输入进行处理，最终得到整个 Decoder block 的输出
4. 上述三层的每一层后都会有残差连接和层归一化操作

在经过若干 block 处理后，输出内容会经过一个全连接层，并得到最终自回归需要生成的文字，并进入下一轮自回归生成循环

#### Decoder block

**掩码多头自注意力**：

将 Decoder 已生成的文本序列经过词嵌入和位置编码后（或其他 Decoder block 的输出）作为该层输入，内部和前面介绍的多头自注意力层计算过程一致

唯一的区别在于，由于当前已生成的文本序列是一次性输入的，为了避免先生成的内容关联到后生成内容的信息，所以引入 **上三角矩阵掩码 M**。该矩阵的对角线以上为 $-\infty$，对角线及以下为 0，防止自注意力关联时某 token 读取到未来 token 的信息。计算过程可以表示为：

1. 输入矩阵 $X$ 首先投影出 $Q, K, V$：
$$Q = X W_Q, \quad K = X W_K, \quad V = X W_V$$
2. 接着计算未归一化的注意力得分 $S = \frac{Q K^T}{\sqrt{d_k}}$，引入一个上三角矩阵掩码 $M$：
$$S_{\text{masked}} = S + M$$
3. 最后通过 Softmax 计算输出：
$$\text{Output} = \text{Softmax}(S_{\text{masked}}) V$$

**编码器-解码器交叉注意力**：

上一层的输出 $X_{mid1}$ 以及 Encoder 输出 $H_{enc}$ 分别输入到这一层进行计算：
- **Query ($Q$)** 来源于 Decoder 本身：$Q = X_{\text{mid1}} W_Q$ （形状为 $N_{\text{target}} \times D$）
- **Key ($K$) 和 Value ($V$)** 来源于 Encoder 的全局特征 $H_{\text{enc}}$：$K = H_{\text{enc}} W_K, \quad V = H_{\text{enc}} W_V$ （形状为 $M_{\text{source}} \times D$）

计算跨界注意力：
$$\text{Output} = \text{Softmax}\left(\frac{Q K^T}{\sqrt{d_k}}\right) V$$
在这里，$Q$ 代表解码器当前“正在翻译写到哪里了”；$K$ 代表编码器对源文本“每个词各自带有怎样的语义标签”；$V$ 代表源文本“每个词包含的实际内容特征”

> [!tip]
> 在这一层中，Encoder 输出仅作为一个只读的可参考数据，最终输出的数据规模取决于上一层输出 $X_{mid1}$

#### Decoder 输出

经过若干 Decoder block 的处理后，还需要将输出转化为下一个需要生成的词

**步骤 1：切片提取 (Slicing)**

不需要管前面 $N-1$ 个词的输出，只需要这个矩阵的最后一行（即刚刚输入的最后一个词对应的 $D$ 维特征向量）抽出来进行后续处理

**步骤 2：语言模型头映射 (LM Head Projection)**

将这个 $D$ 维向量送入一个巨大的全连接线性层。这个线性层的权重矩阵尺寸为 $D \times V_{\text{vocab}}$（$V_{\text{vocab}}$ 是字典总词数），计算后得到了一个长度为 $V_{vocab}$ 的向量。这个向量里的数字可以是任意实数，代表了字典里每一个词成为下一个词的“原始得分”

**步骤 3：Softmax 概率化**

将该向量使用 Softmax 函数进行处理，把它们压缩成 0 到 1 之间的概率分布，且总和为 1

**步骤 4：解码策略 (Decoding / Sampling)**

面对这个概率分布，硬件决定到底选哪个词输出：

- **贪心搜索 (Greedy Search)**：简单粗暴，直接选概率最高的那个词
- **采样 (Top-k / Top-p Sampling)**：为了让生成的文本更有创造性，系统会在概率最高的几个词里按比例随机选取（可以使用 temperature 对随机性进行控制）

自回归生成下一个词后即可进行下一轮循环

> [!tip]
> 使用 Decoder 进行文本生成时，初始（N=1）token 被设置为特殊的起始 token，如 `<BOS>`
> 
> 输入的 N 的 token 进行若干 decoder block 处理后，选择第 N 个 token 进行处理并得到下一个生成的词。当 N=1 时会选择 `<BOS>` 对应的向量生成下一个词 `token1`；N=2 时会选择 `token1` 对应的向量生成下一个词 `token2`

#### Temperature

在大语言模型中，**Temperature（温度系数）** 是控制模型生成随机性（或创造性） 的一个关键超参数。简单来说，它的作用是调整模型预测下一个词时的概率分布形态，从而影响输出结果的多样性与确定性

模型在预测下一个词时，会先为词表中的每一个词计算一个分数（Logit，逻辑值），然后通过 Softmax 函数将这些分数转化为概率

引入温度 $T$ 后，Softmax 公式变为：
$$
P_i = \frac{e^{z_i / T}}{\sum_{j} e^{z_j / T}}
$$
这里的 $z_i$ 是模型输出的原始分数，$T$ 是温度系数
- 当 $T$ 越低（趋近于 0）：$z_i / T$ 会变得非常大。这会导致原本分数最高的那个词，其概率被极度放大（趋近于 1），而其他词的概率趋近于 0
- 当 $T$ 越高（大于 1）：$z_i / T$ 会变小。所有词的得分会被拉平，最高的不再那么突出，低的也不再那么低

一般而言：

| T=0  | T=0.1~0.5 | T=0.7~1.0 | T>1.0 |
| ---- | --------- | --------- | ----- |
| 贪婪解码 | 较为稳定      | 平衡        | 较为激进  |

- 调用外部 API/工具时：推荐 Temperature 强制设为 0。因为工具调用需要严格的 JSON 格式或函数名，不能有随机性
- 基于文本回答时：Temperature 设为 0.7~1.0，发挥模型泛化能力

#### KV-Cache

在真实的硬件部署中，如果按照上述公式直接运行 Decoder，在自回归推理时会遇到严重的**算力浪费**

需要注意的是，在计算第一子层和第二子层时，过去已经生成的那些词的 Key 和 Value 向量是完全不会随着新词的出现而改变的。这会导致每次自回归生成时都会让整个网络对所有 token 进行重复计算，导致大量的性能损失

因此在工业级大模型推理中，硬件会开辟一块 SRAM/HBM 缓冲区，专门用来存储过去所有时间步计算出的 $K$ 和 $V$。每次循环，系统只计算当前新词的 $Q, K, V$（因为掩码多头自注意力层使得前面的 token 不会受到后续 token 影响），然后直接拼接到历史的 **KV Cache** 缓存中进行 Attention 计算

### Decoder-only 架构

#### 架构调整

Decoder-only 架构中删除了 Encoder 部分，同时将 Decoder block 中的交叉注意力层删除。

- 原始 Decoder 块结构 = `掩码自注意力层` + **`交叉注意力层 (Cross-Attention)`** + `前馈神经网络 (FFN)`。
- 现代纯 Decoder 块结构 = `掩码自注意力层` + `前馈神经网络 (FFN)`。
- 当系统剔除 Encoder 时，Decoder 内部负责接收 Encoder 信号的交叉注意力层也被全部删除，这表示在相同的总体参数量预算下模型可以堆叠出更深的纯 Decoder 网络（例如将 12 层 Enc + 12 层原始 Dec，等价转换为约 28 层纯 Dec）

#### 学术界性能验证与路线演进

模型性能的优劣高度取决于**评估范式**（微调 vs. 零样本泛化）。学术界的三项研究界定了两者的适用边界：
- **微调范式（T5 模型，Google 2020）**： 在同等参数规模下，若采用“预训练 + 针对特定任务微调（Fine-tuning）”范式，Encoder-Decoder 架构表现最优。因为 Encoder 的双向注意力机制（Bidirectional Attention）能够无损且极其高效地提取全局上下文特征
- **缩放定律与涌现能力（GPT-3 系列，OpenAI 2020）**： 研究证明，纯 Decoder 架构高度契合**缩放定律（Scaling Laws）**。当参数规模与数据量呈指数级扩大（如达到千亿级别）时，纯 Decoder 模型涌现出了强大的上下文学习（In-context Learning）能力
- **零样本泛化（Wang et al., Google 2022）**： Google 的大规模对比研究《What Language Model Architecture and Pretraining Objective Work Best for Zero-Shot Generalization?》说明：如果目标是构建具备极强“零样本泛化能力（Zero-Shot Generalization）”的通用人工智能，Decoder-Only 架构配合因果语言建模（Causal Language Modeling, 即预测下一个词）是综合表现最优的选择

#### Decoder-only 架构在工程上的优势

- **计算范式极度统一**： Decoder-Only 架构将所有 NLP 任务（问答、翻译、推理）在数学上完全统一为“自回归下一个词预测（Next-token Prediction）”。这种统一彻底消除了源文本与目标文本的数据流割裂，极大简化了分布式训练（如流水线并行、张量并行）的底层算子调度复杂性
- **内存与缓存优化（KV Cache）**： 在推理生成阶段，Encoder-Decoder 需要维护两套独立的内存状态（Encoder 的全局输出特征与 Decoder 的历史缓存）。而 Decoder-Only 架构完全契合单向数据流，结合现代推理加速技术（如 vLLM 的 PagedAttention 和 FlashAttention），硬件层面只需维护单一的连续 KV Cache 池，使得显存带宽（Memory Bandwidth）的利用率达到物理极限
- **超长上下文的架构扩展性**： 现代大模型致力于支持极长的上下文窗口（如 128K 甚至 1M+）。在双体架构中，处理极长文本会导致交叉注意力层的显存消耗呈灾难性爆炸；而在纯 Decoder 架构中，结合旋转位置编码（RoPE）等技术，单向因果注意力更容易实现长度的外推与显存的线性/亚线性优化
